🧭 진행 상황

① 앱 트랙 — quizdeck

  • 데이터 추출 (Phase 0)
    단일 HTML의 5개 JSON 블록을 content/aws/sap-c02/*.json로 무손실 분리. 콘텐츠 디렉토리 규약 확립.
  • Next.js 전환 (Phase 1–2)
    Next 15 App Router 정적 export + React 19 + Tailwind v4 + TS. 퀴즈엔진·개념·다이어그램·검색·맵·PDF 전체 패리티 이식.
  • ProgressStore seam (TDD)
    진도 영속화를 snapshot·async 인터페이스 뒤로 deepen. Progress(durable)/Session(transient) 분리. vitest 14 GREEN. CONTEXT.md + ADR-0001.

② 인프라 트랙 — Synology VPC

  • 정리 + 보안
    옛 Docker 컨테이너 전부 제거(36GB 회수). 원격 접속을 Tailscale로 이전, 인터넷 SSH 노출 폐쇄.
  • L1 — k3s substrate
    커널 4.4 한계로 k3d 탈락 → VMM의 modern-커널 VM k3s-home(커널 6.8)에 k3s. Traefik ingress 동봉.
  • L3 — GitOps 배포
    Dockerfile + kustomize + GitHub Actions + Argo CD. PR#1 머지 → 자동 빌드·배포 → Synced/Healthy.
  • L2 — Cloudflare Tunnel
    myquizdeck.com 구매 → cloudflared(k3s)로 공개. 인바운드 0·홈IP 은닉·HTTPS.
  • 다듬기
    cloudflared까지 git/Argo 편입(완전 선언적), 옛 index.html 제거 + GH Pages 비활성.

📶 진화 로드맵 — 층(Layer)

각 상위 층은 독립적으로 가치 있고, 가치 < 비용 지점에서 멈춥니다. 플랫폼은 워크로드 위에서 진화합니다.

L1
k3s (+ Traefik) 완료
클라우드네이티브 substrate. VM k3s-home, 커널 6.8 / cgroup v2.
L3
quizdeck + Argo CD 완료
첫 워크로드 + GitOps(강제 함수). git push → 자동 배포.
L2

Identity seam + ProgressStore RemoteApi + 외부 DB(별도 DB VM). 기기간 동기화.
L4

메시보다 먼저. 메트릭·로그·대시보드.

운영 방식 · 회복력

  • 배포: git push → CI → Argo 자동 동기화. 수동 단계 없음.
  • 회복력: k3s는 systemd 서비스(재부팅 자동 기동), Argo selfHeal로 드리프트 교정.
  • 보안: 라우터 포트포워딩 0, 홈 IP 은닉, HTTPS 엣지 인증서, 관리는 Tailscale 전용. 터널 토큰은 k8s Secret(레포 밖).
  • GitOps: quizdeck · cloudflared 둘 다 Argo Application(Synced/Healthy).
