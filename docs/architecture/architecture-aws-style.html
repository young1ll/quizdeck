흐름

#요청 경로 (런타임)
1사용자 → Cloudflare 엣지(HTTPS 종료)
2Cloudflare → cloudflared 아웃바운드 터널로 (집 라우터 인바운드 포트 0)
3cloudflared → Traefik ingress (Host 헤더 보존)
4Traefik → quizdeck pod (Next standalone 서버 :3000)
5quizdeck → postgres(db-home) — 세션·인증·JWKS. 5432는 k3s-home(.55)에서만(ufw+pg_hba)
6quizdeck → WordPress REST(클러스터 내부 Service) — 콘텐츠 read(게시본만) + ISR 캐시 — ADR-0025
7WordPress → MariaDB(db-home) — 콘텐츠 저장. 3306은 k3s-home(.55)에서만(ufw)
#GitOps 흐름 (배포)
Agit push → GitHub Actions: pnpm test + Docker 빌드
B이미지 → ghcr.io push + kustomize 태그 bump 커밋 (app=deploy.yml · WP=wp-image.yml, infra/wp/** 변경 시)
CArgo CD가 main 감시 + ghcr 이미지 pull — App ×5: quizdeck · cloudflared · backup · argocd-server-expose · wordpress
DArgo가 quizdeck 배포(rollout) — Synced/Healthy · CI가 공개 /api/health sha 폴링으로 롤아웃 확정
·운영 흐름 (관리 · 백업)
T1관리자 → Tailscale → VM SSH (k3s-home=100.81.230.113) — 인터넷 SSH 없음
T2관리자 → Tailscale → argocd.myquizdeck.com — grey-cloud DNS가 tailnet IP를 가리켜 인터넷 도달 불가, Traefik이 DNS-01 LE 인증서로 종단 → argocd-server(--insecure). 검증: verify-argo-exposure.sh 3단언 (ADR-0020)
T3편집자 → Tailscale → wp.myquizdeck.com — ADR-0020 패턴 그대로(grey-cloud DNS + IngressRoute + DNS-01 LE). WP admin(콘텐츠 저작)은 인터넷 도달 불가 — WP 최대 공격 표면의 구조적 차단 (ADR-0025)
WWP 편집 발행(게시본 영향 시에만) → 웹훅 /api/revalidate-content(공유 토큰, 클러스터 내부) → Next ISR revalidate — 편집이 수 초 내 서빙에 반영
BKCronJob db-backup(ns quizdeck, 03:00, pg_dump -Fc) + wp-db-backup(ns wordpress, 03:20, mariadb-dump) → mc pipe → R2 quizdeck-backups (30일 롤링). 오프사이트 = 박스 loss 생존 (ADR-0021·0025)
